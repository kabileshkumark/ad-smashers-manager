## Release Summary

Version 1.0 is the first formal SDLC-managed release for AD Smashers Manager.

The formal managed release and the technical app/cache build are aligned at `1.0`. The npm package metadata remains valid semantic version `1.0.0`, while the user-facing PWA version, service-worker cache, manifest, and asset versions use technical build `1.0`.

## Included Change

- Cloud sync hardening: pending cloud restore now tolerates null or malformed change journals from older saves.
- Attendance control: extra confirmed players remain included after attendance becomes manually controlled, and their payment records stay synchronized.
- Version alignment: app shell, service worker, manifest, and config are synced to technical build `1.0`.
- Regression coverage: added coverage for null pending cloud change journals and extra confirmed players under manual attendance.

## Manual Smoke Checklist

- Sign in works.
- Sessions load.
- Session save/reload works.
- Payments and balances load.
- App version/update behavior works.

## Rollback

- Keep the pre-deploy live data backup.
- Identify the last known good Git commit/build.
- Rebuild and redeploy the prior version if smoke testing fails.
- Restore backup only if release causes data corruption.
