## Release Summary

Version 1.0 is the first formal SDLC-managed release for AD Smashers Manager. Technical build 1.0.3 is a compatible payment-accounting correction within Version 1.0.

The app shell, npm package, service-worker cache, manifest, and asset versions use technical build `1.0.3`. Jira and Confluence retain the formal release name `Version 1.0`.

## Included Changes

- Payment-group Credit ownership: post-game overpayment Credit remains entirely with the payer.
- Credit terminology: positive post-game remainder is shown as Credit; intentional Advance remains in the Advance section.
- Payer Credit settlement: remaining payer Credit reduces the payment group's cash payable amount and is consumed before new cash.
- Reversible audit: Credit use is recorded on the group-payment transaction and restored when that transaction is deleted.
- Regression coverage: the exact AED 122 stored Credit, AED 40 payer due, AED 82 remaining Credit, and AED 40 member due scenario is covered, along with Credit-plus-cash and intentional-Advance isolation cases.
