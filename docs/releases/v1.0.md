## Release Summary

AD Smashers v1.0 is the first formal SDLC-managed release for AD Smashers Manager.

The app already uses technical build version `1.0.84` from earlier development iterations. For release management, `v1.0` is the first formal release line and `1.0.84` is the technical app/cache build.

## Included Change

- Cloud sync hardening: pending cloud restore now tolerates null or malformed change journals from older saves.
- Version alignment: app shell, service worker, manifest, and config are synced to technical build `1.0.84`.
- Regression coverage: added coverage for null pending cloud change journals.

## Required Before Deploy

- Export or back up live AD Smashers data.
- Review shared Firestore rules in Firebase project `home-kaish`.
- Confirm no Healthy Lifestyle App rules/files are deployed accidentally.
- Review dirty Git state and decide what belongs in the v1.0 release commit.
- Run live smoke test after deploy.

## Manual Smoke Checklist

- Sign in works.
- Sessions load.
- Session save/reload works.
- Payments and balances load.
- App version/update behavior works.
- Mobile/PWA navigation works.

## Rollback

- Keep the pre-deploy live data backup.
- Identify the last known good Git commit/build.
- Rebuild and redeploy the prior version if smoke testing fails.
- Restore backup only if release causes data corruption.
