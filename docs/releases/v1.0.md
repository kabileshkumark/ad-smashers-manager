## Release Summary

Version 1.0 remains the formal SDLC-managed release for AD Smashers Manager. Technical build 1.0.5 is a compatible presentation and navigation patch within Version 1.0.

The app shell, npm package, service-worker cache, manifest, and versioned assets use technical build `1.0.5`. Jira and Confluence retain the formal release name `Version 1.0`.

## Included Changes

- Dashboard feedback: selecting the top or sidebar brand from another page paints the existing loading overlay before Dashboard rendering begins and blocks duplicate navigation while pending.
- Player Balances: payment, copy, and history controls remain aligned at the top-right with responsive square sizing.
- Payment Groups: payment amount starts blank; amount plus five actions stay in one responsive top-right row.
- Advance History: the balance chip and reversal control remain top-right while deduction entries use the full content width below the transaction header.
- Traceability: `FR-ACC-004`, `FR-PAY-002`, `FR-PAY-008`, `FR-PAY-009`, `NFR-UX-001`, and their mapped regression cases were updated.
- Existing financial behavior from technical build 1.0.4 is unchanged. This patch does not alter ledger calculations, Auth, persistence, Firestore rules, or Firebase data.
